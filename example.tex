\hsgsection[Symmetric, asymmetric, session keys]{Encryption Mechanisms}

\begin{frame}
\frametitle{Two-column content example}
\framesubtitle{Illustration}

\begin{columns}[T]
  \column{0.5\textwidth}
    \textbf{\color{HSGgreen} Symmetric encryption}
    \begin{itemize}
      \item Single shared key
      \item Fast, suitable for bulk data
      \item Key distribution is the hard part
    \end{itemize}
  \column{0.5\textwidth}
    \textbf{\color{HSGgreen} Asymmetric encryption}
    \begin{itemize}
      \item Public / private key pair
      \item Solves key distribution
      \item Orders of magnitude slower
    \end{itemize}
\end{columns}

\vspace{1em}
In practice, \hsgemph{TLS} combines both: asymmetric cryptography establishes a
shared session key, which is then used with a symmetric cipher for the data stream.
\end{frame}

\begin{frame}
\frametitle{Summary}

\begin{itemize}
  \item Symmetric cryptography is fast; asymmetric solves key distribution.
  \item \hsgemph{Certificates} bind public keys to identities; \hsgemph{TLS} uses
        them to bootstrap trust.
  \item Diffie-Hellman gives forward secrecy even if long-term keys leak later.
\end{itemize}
\end{frame}
